# 100 Days of Python Bootcamp - Days 1 to 16

This repository documents the first stage of my 100 Days of Code: The Complete Python Pro Bootcamp journey. It covers the transition from absolute Python basics to writing small but complete programs that solve real problems, handle user input, and introduce the mindset behind software development.

The value of this repo is not just that it contains exercises. It shows the progression from learning syntax to building logic, from simple scripts to interactive games, and from isolated concepts to reusable problem-solving patterns. For recruiters, that progression matters because it demonstrates consistency, curiosity, and the ability to turn new concepts into working code.

## What This Repo Covers

- Python fundamentals: variables, strings, input/output, and basic arithmetic.
- Control flow: conditionals, nested logic, and boolean reasoning.
- Loops: `for` loops, `while` loops, repetition, and iteration.
- Functions: reusable logic, parameters, return values, and scope.
- Data structures: lists, dictionaries, nested collections, and iteration over data.
- Modules and randomness: using built-in libraries to add behavior and unpredictability.
- Debugging and problem solving: reading errors, tracing logic, and fixing broken code.
- Mini-projects: practical programs that combine the above concepts into complete apps.

## Learning Progression

The repo follows a very natural development path:

1. Days 1-2: Core Python and math
	- Printing, input, variables, type conversion, and numerical operations.
	- Projects such as the tip calculator introduced the idea of taking user input and producing a useful output.

2. Day 3: Conditionals and logic
	- `if` / `elif` / `else`, nested decisions, and logical operators.
	- Projects like Treasure Island and Python Pizza showed how branching logic creates interactive experiences.

3. Day 4: Randomness and lists
	- Working with the `random` module, list indexing, and error handling.
	- Rock, Paper, Scissors introduced game flow, validation, and comparing outcomes.

4. Day 5: Loops and iteration
	- `for` loops, range-based loops, and summing or searching through data.
	- The password generator was a strong step toward building something practical from repeated operations.

5. Day 6: Functions and code reuse
	- Breaking problems into smaller pieces, naming behavior clearly, and reducing repetition.
	- This is where the code starts to look like software rather than a single-use script.

6. Day 7: Hangman-style thinking
	- Combining loops, conditionals, and state tracking in a larger game loop.
	- This kind of project teaches how to manage user progress over time.

7. Day 8: Functions with arguments
	- Parameters, positional versus keyword arguments, and modular design.
	- Caesar Cipher is a classic example of transforming data through reusable logic.

8. Day 9: Dictionaries and nested data
	- Mapping names to values, storing structured information, and iterating through records.
	- The blind auction project is especially useful because it introduces multi-user input and decision making.

9. Day 10: Functions with return values
	- Returning results instead of only printing, which is a major step in writing maintainable code.
	- The calculator project showed how to organize a multi-step application around reusable functions.

10. Day 11: Capstone-style Python practice
	- Stronger emphasis on combining previously learned concepts into a complete challenge.

11. Day 12: Scope and namespaces
	- Local vs global variables, constants, and debugging scope-related behavior.
	- This is an important milestone because it deepens understanding of how Python manages state.

12. Day 13: Debugging and problem solving
	- Reproducing bugs, reading errors carefully, and using a debugger effectively.
	- These skills are essential in real engineering work, not just in bootcamp exercises.

13. Day 14: Higher or Lower project
	- A larger game loop with score tracking, reusable formatting, and comparison logic.
	- This project reflects the shift from classroom exercises to something closer to a portfolio piece.

14. Day 15: Coffee Machine project
	- Functions, state management, resources, transactions, and menu-driven application flow.
	- One of the most practical projects in the early bootcamp because it behaves like a small real-world system.

15. Day 16: Object-oriented programming introduction
	- Classes, objects, constructors, and methods.
	- This is the bridge from procedural thinking to designing code around entities and behavior.

- I learned how to break a problem into smaller steps.
- I built interactive programs instead of only writing isolated snippets.

- `Day 1/` and `Day 2/`: input, variables, math, and first mini-projects.
- `Day 3/` and `Day 4/`: conditionals, randomness, lists, and game logic.

## How To Run

1. Make sure Python is installed.
2. Open the day folder or project you want to run.
3. Execute the script from the terminal, for example:

```bash
python "Day 2/Tip Calculator Project/solution.py"
python "Day 3/Treasure Island Project/solution.py"
python "Day 15/Coffee Machine Project/main.py"
```

Some exercises are designed as guided practice files, while others are full projects. A few projects may rely on simple console input, so they are meant to be run interactively.
